A Content-Addressed Adaptive Knowledge Substrate

N. Joven · Feb 2026 · CC0 · Paper · Implementation (Rust)

The problem is the substrate

RAG systems chunk documents, embed them as vectors, and retrieve by similarity. This destroys relational structure, uses a lossy proxy for relevance, and rebuilds context from scratch every session. Fine-tuning bakes knowledge into opaque weights. LLMs cannot self-correct without external feedback. The fix is not a better model — it is a better substrate.

§2.1 Content-Addressed Nodes

Every node hashed from its content + dependencies. Identity = hash. Dedup is automatic. The entire knowledge state resolves to one root hash.

§2.2 Depth Scoring

Live scores govern traversal depth per node. Inputs: recency, information gain, structural centrality, query relevance. Cheap enough to run thousands of times per cycle.

§4 Structural Provenance

A conclusion's hash is computed from its premises' hashes. Change any upstream fact → the derived hash changes. Provenance isn't metadata — it's identity.

§2.5 Fixed-Point Convergence

Reasoning terminates when successive cycles produce the same root hash. No self-assessment needed — convergence is hash equality.

Tier 1 — Free

Hash compare, threshold check, delta propagate. Run always.

Tier 2 — Moderate

Subgraph traverse, consistency check, merge candidates. Run when flagged.

Tier 3 — Expensive

Deep traversal, LLM reasoning, durable writes. Run when earned.

§8 — How it scales: demand-driven consistency

Consistency is demand-driven, not supply-driven. Nodes carry no propagation cost while unqueried. Staleness is detected at read time via hash comparison — the query itself is the consistency mechanism.

Hot
Recently queried. High depth score. Epistemically current. Expensive.
Cooling
Score decaying. Still cached. Will reveal staleness on next read.
Cold
Unqueried. Zero cost. Frozen potential. Reconciles on contact.
The graph doesn't get more expensive as it grows. It gets colder. And cold is free.
